# 2026-03-20 Conda 环境调整总结

## 本次目标

将项目的本地运行环境从 `venv` 方式切换为 `conda` 方式。

## 本次完成内容

1. 新增 `environment.yml`，定义项目推荐的 Conda 环境。
2. 将 `README.md` 中的环境创建说明改为 `conda` 流程。
3. 保留 `pip install -e .[dev]` 的安装方式，以便继续使用标准 Python 打包结构。
4. 准备移除之前临时创建的 `.venv`。

## 设计取舍

- 这里选择 `python=3.11`，而不是继续使用 `3.13`。
  原因是从兼容性和生态稳定性看，`3.11` 更适合作为学习和示例环境。
- 依赖安装仍通过 `pip` 完成。
  原因是 `langgraph` 及其相关生态通常以 PyPI 分发为主，Conda 更适合负责 Python 版本与环境隔离。

## 风险与限制

- `environment.yml` 使用了本地可编辑安装 `-e .[dev]`，因此应在项目根目录执行 `conda env create -f environment.yml`。
- 如果你的本机已有同名环境，需要先删除旧环境或手动更新。

## 下一步建议

1. 在 `langgraph` Conda 环境中继续验证 CLI 和测试。
2. 后续再进入 `MessagesState` 或 LLM 节点学习。
